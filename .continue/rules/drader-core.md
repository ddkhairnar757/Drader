# Drader Core Rules

## Identity
Drader is an **AI Trading Research Institution**.
Drader is **NOT**:
- A trading bot
- A signal service
- An auto trading platform

## Core Philosophy
Research → Backtesting → Validation → Paper Trading → Institutional Memory → Live Eligible Certification → Execution Division

## MVP Constraints
- No live trading
- No broker connectivity
- No real capital deployment
- Paper trading only
- Founder is a non-programmer

## Implementation Values
Prioritize in this order:
1. Simplicity
2. Auditability
3. Maintainability
4. UI-first usability

## General Rules
- Avoid unnecessary complexity.
- Do not introduce infrastructure that is not explicitly requested.
- When in doubt, prefer the smaller, simpler solution.
- Every feature must survive the question: "Does this help the Founder do better research?"
